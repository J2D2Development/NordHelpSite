    Working with the Directory: {{directorypanel.msg}}

    The resident directory is a database of information about the residents in your community. It is typically one of the most popular features on an HOA site. Since the feature can be secured (i.e. made available only to other residents, or only to specific people) and since a resident can alter their information – remove email addresses or phone numbers – security of private information is assured.

    Normally, the initial information is uploaded from a list you provided at start up time, so maintenance is only required when residents move in or move out. However, if you don't have an initial database of residents, individual entries can be easily added.

    There are several options for controlling the display of information on the web site directory screen. To access the resident’s directory, select “Features” on the administrator’s panel and you'll see a row for "Owner/Resident Directory"

    Click the 'edit' link for the owner/resident directory feature and you'll see a screen similar to the one below:

    Site Title – Enter the title that you would prefer to use on the directory display (e.g. Our Community, Community Listing, etc.).

    Feature Admin – If you wish to assign responsibility for directory maintenance to another user, you may select a feature administrator from the drop-down list. Only those users who have the “feature administrator: option selected in their user records are displayed here. The directory administrator can update the directory directly from the site by using the “manage” button on the directory screen.

    Require User Authentication – If you want to secure the display, select “Yes”, requiring a user logon. Select “No” if you do not. Since the directory displays information that may be considered private, we recommend that you set this option to “yes.”

    Set Group Authentication Limit – If you want to make the directory display on the site available only to members of one or more of your defined User Groups, change the “Set User Group Authentication Limit?” from a “no” to a “yes”. You can then select any of the User Groups you have defined. As an example, if you wanted the directory available only to members of the Board, and the “Board Members” group was defined in the “User Group Feature,” you would set the User Groups authentication option to a “yes” and select the “Board Members” group. For more information on setting up User Groups, see the User Groups guide (coming soon).

    Threshold for search/index mode – For very large directory listings you can elect to display the directory with an “index,” requiring the user to select a letter (last name) before displaying the entries. Normally, you will want to display the entire directory, but you can set the threshold that causes the index to appear here.

    Allow user to update directory - Select “yes” if you want to allow users to update their own directory records. If you select “yes” and the user record is tied to the directory record by means of a common property address (the Home record), then your users will be able to select the “Your Entry” button on the directory display. If you select “no” they will not have the ability.

    Update user record for directory entries – Select “yes” if want to allow the changes a user makes to their directory record to update the information in their User record. You can select “yes” which enables the automatic updates; “no” which disables the automatic update; or, “notify” which enables the update and sends an email to the administrator notifying them (you) of the changes that have been made in the Directory and used to update the User record. This option is only available if the “Allow user to update directory…” option is set to “yes.”

    Use Building/Section - If your community is subdivided into buildings or sections, you can optionally sort the directory display by those buildings or sections. To do that, select “yes” here. When a resident views the directory and selects the “By Address” option, the listing will be that sequence. Note: the building/section must be in the Home record for this option to work. You can also force the system to display an index of the buildings or sections by lowering the “threshold” to a number lower than the actual number of home records.

    Include non-resident owner - If you would like the directory listing to include the name and addresses of any non-resident owners, select “yes,” if not select “no.”

    Display non-resident owner’s off-site - If you would like to display the out-of-community addresses of any non-resident owners, in addition to their in-community address, select “yes”. If not, select “no.” If you choose “yes” an additional line “Off-Site Address” will be displayed for all entries, and the actual off-site address will appear below this line for any non-resident owners.

    Display lot numbers - This option allows you to include the lot number on the display. It is only valid if lot numbers were captured in the Home record.

    You can define your own labels for four additional, multi purpose fields. The first two appear in the “Home” section of the directory. The second two appear in each of the “entry # information” sections.

    General Custom Field 1 – Enter the field label (optionally) for field 1. This option will then appear when you edit any individual directory entry.

    General Custom Field 2 – Enter the field label (optionally) for field 2. This option will then appear when you edit any individual directory entry.

    Per Entry Custom Field 1 – Enter the field label (optionally) for field 1. This option will then appear when you edit any individual directory entry listing.

    Per Entry Custom field 2 - Enter the field label (optionally) for field 2. This option will then appear when you edit any individual directory entry listing.

    Introduction Text - Any text entered here will appear above the main directory listing in the user interface. You can use this to provide details about the directory or notices specific to that section.

    If you already have a list of residents and their addresses in electronic format (e.g. an Excel Spreadsheet or Access Database) and can produce a comma-separated values (.csv) file with the information, you do not have to create the users, directory, and homes records manually. You can upload the information directly into the system. Directions for uploading are below, but formatting can be tricky, so contact your customer support manager if you need a hand and they'll be glad to help.

    The process is not complicated.
    1) Create the .csv file
    2) Upload the information into the Directory (creating the Homeowners Directory and Homes List at the same time)
    3) Create the User Ids from the uploaded Directory information (optional, but recommended).

    You should upload the directory Information first and make sure you have a successful load. This will create the Homeowner's Directory and the Homes list. Once this file has been successfully uploaded, you can create the user id's from it - immediately or later. Generally, only Primary Resident/Owner data is populated via upload. Additional household members (e.g. children, roommates) can be added by you or the homeowners after the initial load.

    Note: This process is designed to be a one-time, start-up data load process. Performing updates with this process is possible, but not recommended. Instead, updates should be performed directly in the Directory.

    Creating the .csv file:
    Prepare your data file with 4 or more columns of data. Do not include any column header rows in your file. Check – and edit – the file for consistency and accuracy (capitalization, spelling, missing information, etc.). Each Directory entry consists of a "home record" (the address of the home in the community) and information about the Primary Resident and/or Owner of that home (name, phone number, email address, etc.).

    An example file might look like:

    A “Home record” is made up of as few as 2 fields and as many as 4, which form the postal address for the home. These 4 fields must be unique for each home.

    The 4 fields are:

    • Street Number (required)
    • Street Name (required)
    • Number Suffix (optional)
    • Unit Number (optional)

    In addition to the address fields, two other fields are available for further classification or identification of the homes in your community:

    Building/Section - Classifies the homes using a Building Name (e.g. in a condo complex) or a Section Name (e.g. in a large community)

    Lot – Classifies the homes by the Lot Numbers, which are usually assigned by the builder during development.

    The Street Number and Street Name are the only required fields and will typically be the only fields needed in a community of single-family homes. Condo and apartment complexes may need to use the Unit and/or Building/Section information as well.

    There are 2 required fields for the Primary Resident and/or Owner information (Last Name and First Name). In addition, there are 13 optional fields that you may choose to populate:

    Entry Type - specifies if the person is a (r)esident, an (o)wner, or (b)oth. If you do not provide this information in your upload file, all directory entries will be set to "b" for both.

    Phone Number - the personal phone number for the resident or owner. If you do not provide this field in your upload file, it will be left blank.

    Email Address - an email address for the primary resident and/or owner. If you do not provide this field in your upload file, it will be left blank. Note: email addresses are a critical part of making best use of your web site. We strongly suggest that you solicit and load email addresses as part of the upload. When you later create the user ids from this information, the system will send all the users with email addresses on file a start up announcement giving them their user id and password information.

    Land Line - the phone number physically located within the home, it is not intended for cell phone numbers. If you do not provide this field in your upload file, it will be left blank.

    Custom Home Field 1 – Referred to on the directory edit screen as “General Custom Field 1”. One of the two available custom fields that you can define for the home (e.g. Year Joined, Number of Children. etc.). If you do not provide this field in your upload file, it will be left blank. You have the ability to set the label for this field in the edit screen for the directory feature.

    Custom Home Field 2 - Referred to on the directory edit screen as “General Custom Field 2”. One of the two available custom fields that you can define for the home (e.g. Number of Pets, Parking Slot Number, Pool Key, etc.).If you do not provide this field in your upload file, it will be left blank. You have the ability to set the label for this field in the edit screen for the directory feature.

    Custom Per Entry Field 1 – Referred to on the directory edit screen as “Per Entry Custom Field 1”. One of the two available custom fields that you define for each resident of the home (e.g. Marital Status, Cell Phone Number, etc.). If you do not provide this field in your upload file, it will be left blank. You have the ability to set the label for this field in the edit screen for the directory feature.

    Custom Per Entry Field 2 – Referred to on the directory edit screen as “Per Entry Custom Field 2”. One of the two available custom fields that you define for each resident of the home (e.g. Work Phone Number, etc.). If you do not provide this field in your upload file, it will be left blank. You have the ability to set the label for this field in the edit screen for the directory feature.

    Outside Address 1 – The non-resident owner’s outside mailing street address. If you do not provide this field in your upload file, it will be left blank.

    Outside Address 2 - The non-resident owner’s outside mailing street address (use this field for apartment number, etc). If you do not provide this field in your upload file, it will be left blank.

    Outside City – The non-resident owner’s city. If you do not provide this field in your upload file, it will be left blank.

    Outside State - The non-resident owner’s state. If you do not provide this field in your upload file, it will be left blank.

    Outside Zip - The non-resident owner’s postal/zip code. If you do not provide this field in your upload file, it will be left blank.

    The sequence of the data in the file must be as follows (see image below of the checkboxes for another example of the order):
    Number, Suffix, Street Name, Unit, Building/Section, Lot, Last Name, First Name(s), Entry Type, Phone Number, and Email Address.

    In this example, we are not using the Suffix, Unit, Lot, or Entry Type so they are not in our data file. We are using the Number, Street Name, Building/Section, Last Name, First Name(s), Phone Number, and Email Address and they are in the correct sequence for uploading.

    To Perform the Upload:
    Select “Features” from the Admin Page, then select “Manage Directory” from the Features Page. Finally, select “Upload Directory Entries from a file”. You will see instructions for performing the upload. Scroll to the bottom of the screen and you will see the Upload Form. In that form, select the columns that are present in your data file, and enter the data file name by selecting the “Browse” button and locating the file on your computer. The sequence of the columns in your data file must correspond to the sequence of the checkboxes in the Upload Process form.

    If you have added some directory records manually, for practice as an example, it is a good idea to delete them before you upload the new information. This will insure that all fields are populated with consistency. You can do that by checking the “Delete…” box (shown above) on the screen before you upload the information.

    The file will be uploaded to the server, the contents will be checked for proper format, and a test load will be performed. The results of the test load will be shown and, if you are satisfied that the data is being loaded correctly (data is in the correct columns, all the records loaded, etc), select "Confirm Load" to perform the permanent load. If something does not look right, select "Skip Load" to skip the permanent load, make any needed changes to the data file, and try again. Again, formatting can be tricky, so contact us if you need any help.

    Creating the User Ids and Passwords
    If you are still working on the site and are not yet ready to announce it, you do not have to create the user ids and initial passwords until you are. Since this process will send an email announcement and log on information to all residents with email addresses on file, you might want to try to collect additional email address in the interim. When you get an email address, enter it in the Directory for that resident.

    Once you are ready to make the grand announcement, select “Users” from the Admin Panel and “Auto Generate User Accounts from Directory Entries.”

    Please Note: If you have added users manually, you should consider deleting them before generating the user accounts. Do not delete your own record. You may have to go into the records that you chose not to delete and “clean them up” after this generation. The “Home” records, as an example, will not be populated. If you've already created some home records and assigned them to existing user accounts, and that home record matches one in your upload file exactly, the original user account will remain intact.

    Once you click the link, you'll see more instructions above a button to start the process. Note the checkbox is already selected- this is your only chance to automatically send all users their email/password during this initial creation process. We recommend leaving it checked, but you can also always resend individual user account info via the 'users' tab on your admin panel.

    This process is also designed as a one-time, start-up only process. Changes , additions, or deletions of user records after this initial load are performed one-at-a-time via the 'users' tab in your admin panel's main left hand menu

    To edit or create an entry, click the 'manage' link for the owner/resident directory on the main features screen. You'll see a listing of any existing entries. There is also a link to create a new entry. Click that if you want to create a brand new directory entry.

    The alphabetical last name letter listing is the default view, but there are two other views available by selecting the links at the top. You can display a listing of all residents in alphabetical sequence by selecting “Manage All Directory Entries” or you can display a listing of all streets by selecting “Go to Entries by Street List”.

    From any of the views, locate the specific directory entry that you want to modify and select “edit” for that record. The screen below is the top portion of the directory maintenance screen.

    Home – By definition, entries in the residents directory are residents or non-resident owners of property in the community and each entry defines a single property. For owners, all entries should be attached to a home record. The process is simple; just select the home in the drop down menu. If you don't see the home in the list, there are two possible options: You've already created a listing for this address, or you haven't created the home record yet (done via the 'homes' tab on the admin panel's main left hand menu).

    Land Line – Enter the phone number (optional) as you want it displayed, e.g. (123) 555-1212, or 123-555-1212.

    General Custom Field 1 – Enter a value (optional) corresponding to the field definition that you have established. Note that this, and all custom fields, will only appear if you've created a title for it on the main directory edit screen (see the 'Configuration' tag at left for details)

    General Custom Field 2 – Enter a value (optional) corresponding to the field definition that you have established.

    Entry 1 Information

    Entry Type – Select “Non-owner Resident” for renters; select “Non-Resident Owner” for those owners that do not live at the property (typically, Landlords); select “Resident Owner” for owners who do live at the property (this is the most common designation). When you select “Non-resident Owner” new fields will be displayed for entry of the out-of-community address as shown in the example screen on the prior page. You should fill in these fields manually.

    User ID – If User IDs for residents at this Home address have already been established in the User record, you can locate the User ID for this person by using the drop down menu button. This is especially important in Entry 1 because it allows that person to modify their directory entry from the directory display, using the “Your Entry” button.

    Name – Enter the name of the resident in the appropriate name fields. The directory will display it exactly as entered so you should enter them with leading capital letters (e.g. “George” – not “george”).

    Date of Birth – Enter the date of birth (optionally) in the yyyy-mm-dd format. Alternatively, you click on the small calendar icon and navigate to the date of birth by using the year 􀃅 and month 􀃅 links at the top of the display.

    Phone – If you need to enter an additional phone number, enter it (optionally) here. Enter it as you want it displayed, e.g. (123) 555-1212, or 123-555-1212. If you have given the user the option, they can remove the phone number – or update it – in their directory record.

    Email Address – Enter the email address (optionally) here. Pay attention to the case of the address – some email services are case sensitive. If you have given the user the option, they can remove the email address – or update it – in their directory record. Email addresses on the directory display page become email links, allowing residents to communicate with each other conveniently.

    Per Entry Custom Field 1 – Enter a value (optionally) corresponding to the field definition that you have established. See the configuration tag at left for more information.

    Per Entry Custom Field 2 – (in this example, not set up); Enter a value (optionally) corresponding to the field definition that you have established. See page 7 for more information.

    When you have completed the information in this section, select “Save and stay here.” When you do that, new field are available for entry of additional residents.

    Entry 2 Information

    These additional entry sections are used to record the information for other residents at this address. Typically, Entry 2 Information is used for a spouse, significant other, or co-owner. Entries 3 and beyond can be used for children, mothers-in-laws, etc. depending on your requirements. The information in these sections is entirely optional. Only the first two entries will be displayed on the web site’s directory screen. Others can be displayed by selecting the “detailed listing” option.

    Entry Type – Select “Non-owner Resident” for renters; select “Non-Resident Owner” for those owners that do not live at the property (typically, Landlords); select “Resident Owner” for owners who do live at the property (this is the most common designation). When you select “Non-resident Owner” new fields will be displayed for entry of the out-of-community address as shown in the example screen on the prior page. You should fill in these fields manually.

    User ID – If User IDs for residents at this Home address have already been established in the User record, you can locate the User ID for this person by using the drop down menu button.

    Name – Enter the name of the resident in the appropriate name fields. The directory will display it exactly as entered so you should enter them with leading capital letters (e.g. “Linda” – not “linda”).

    Date of Birth – Enter the date of birth (optionally) in the yyyy-mm-dd format. Alternatively, you click on the small calendar icon and navigate to the date of birth by using the year 􀃅 and month 􀃅 links at the top of the display.

    Phone – If you need to enter an additional phone number, enter it (optionally) here. Enter it as you want it displayed, e.g. (123) 555-1212, or 123-555-1212. If you have given the user the option, they can remove the phone number – or update it – in their directory record.

    Email Address – Enter the email address (optionally) here. Pay attention to the case of the address – some email services are case sensitive. If you have given the user the option, they can remove the email address – or update it – in their directory record. Email addresses on the directory display page become email links, allowing residents to communicate with each other conveniently.

    Per Entry Custom Field 1 – Enter a value (optionally) corresponding to the field definition that you have established.

    When you have completed the information in this section, select “Save and stay here.” When you do that, new fields become available for entry of additional residents. If later you want to delete the information in one of the sections, simple select the “Delete Entry #” button on the left side.

    Click the 'Pictures' tag at left to see details on the final section of the edit screen- adding a picture to a directory listing.

    At the bottom of the screen when editing any individual directory entry, you'll see a section for adding/editing an image:

    Directory Picture – may be used to display a picture of residents or their home. Pictures entered here are part of your total disk space allocation. If you choose to use this feature, please make sure your pictures are very small – ideally under 10kb. The directory listing displays all pictures 250 pixels wide regardless of their actual size. Pictures entered here are stored in a different folder than your other site images and can only be uploaded from this panel.

    If you have already uploaded the picture, you may select it in the “Select from list” drop-down. This is useful if you have made an error in assigning the photo or want to change the photo periodically. New pictures are uploaded by entering “comments” (e.g. The Allens) and navigating to the appropriate picture on your local PC by selecting the “Browse” button. Once you have located the picture, select “Save and stay here” or “Save and return to list”.

    Your residents can upload their own pictures by selecting “My Entry” from the main directory screen. Since most of your residents will likely not understand how to make their picture suitable for the site, it is a good practice to perform the upload yourself.

    To remove (not delete) a picture, select “No picture selected” from the drop-down menu. Deletion of pictures from this folder requires contacting your support provider.

    For more information on managing picture sizes, please see the Prep tag in the Images guide.

    When your users log in to view the resident directory, they will see a screen similar to this:

    This is the default listing of all directory entries. The first 2 entries for each home will be shown. Clicking on the main last name of any entry (or address, if you've sorted by address) will open the expanded view:

    This will show all info entered for this address including extra people, images, landline, etc. You can click the 'back to listing' link in the bottom right to go back to the main view.

    The directory also has some menu options (if you're using the vertical menu option, they'll be down the left as shown below, if you're using horizontal, they'll be across the top).

    There are options for sorting (By Name, by Address). The button titled Your Entry allows a user to edit their own entry. Note that this capability depends on how you've set up your directory (you can prevent this access on your main directory edit screen). Also, only the first person listed in a directory entry has the capability to edit the entry (to prevent different people overwriting one entry). The Contact and Manage links are for the feature admin (if you've assigned one).

    There is also a search function

    Enter the text you want to search and hit the enter key or search button. When you're done, you can hit the 'reset' link (not shown) to go back to the main directory view.
